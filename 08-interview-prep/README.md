# 08 — Interview Prep

> **Before this:** Complete [07-ai-age](../07-ai-age/ai-dev-workflow/README.md) first. You should have a deployed, working MERN job board before preparing to talk about it.

You've built a real full-stack application. Now you need to be able to talk about it — and about software in general — in a way that earns a job offer.

Technical interviews don't test whether you memorized facts. They test whether you *think like a developer*: whether you understand the tradeoffs behind your decisions, whether you can reason through problems you haven't seen before, and whether you can communicate clearly under mild pressure. This module gives you a structured way to prepare for each dimension interviewers probe.

## How to Use This Module

Each topic follows the same structure as the rest of this curriculum: Mental Model first, then code, then an exercise. The difference here is that **the exercise is always spoken, not typed**. Interview prep is performance training — you need to say things out loud, to another person or to a recording, not just read them silently.

Work through each topic in order. After each one, spend 20 minutes answering the exercise prompt out loud as if you are in an actual interview. Record yourself if you can.
