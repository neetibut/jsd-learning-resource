## The Mental Model

React is a library for building user interfaces from components. A component is a function that takes some data (called props) and returns a description of what to display. React takes that description and updates the DOM to match. When the data changes, React re-runs the function and figures out the minimum set of DOM changes needed to make the display match the new data. This is the core idea: you describe what the UI should look like given the current data, and React handles the how.

The shift from DOM manipulation (Week 4) to React is significant. In Week 4, you wrote code that said 'find this element, set its text to this value.' In React, you write code that says 'given this list of jobs, display them like this,' and whenever the list changes, the display updates automatically. You stop thinking about DOM operations and start thinking about data and the components that display it. The job board's job list becomes a `<JobList jobs={jobs} />` component — clean, reusable, predictable.

JSX is React's syntax for describing UI. It looks like HTML inside JavaScript, but it is neither — it compiles to `React.createElement()` calls. The practical rules: use `className` instead of `class` (because `class` is a reserved word in JavaScript), use `{expression}` to embed any JavaScript value, and always close self-closing tags (`<img />`, `<input />`). Props flow downward from parent to child — a parent passes data to a child via props, and the child cannot modify those props directly.

## The Code

```jsx
// frontend/src/App.jsx — Week 9: React job board

// --- 1. A single job card component ---
// Props are the function's parameters. This component is pure:
// given the same job, it always renders the same output.
function JobCard({ job }) {
  return (
    <li className="job-card">
      <h2 className="job-title">{job.title}</h2>
      <p className="job-meta">
        {job.company} &middot; {job.location} &middot; {job.type}
      </p>
      {job.description && (
        <p className="job-description">{job.description}</p>
      )}
    </li>
  );
}

// --- 2. The job list component ---
// Receives the jobs array as a prop and renders a JobCard for each.
function JobList({ jobs }) {
  if (jobs.length === 0) {
    return <p className="empty-state">No jobs found.</p>;
  }

  return (
    <ul className="jobs-list">
      {jobs.map(job => (
        // key helps React track which items changed during re-renders.
        // Always use a stable, unique ID — not the array index.
        <JobCard key={job._id} job={job} />
      ))}
    </ul>
  );
}

// --- 3. The root App component ---
// Holds the static job data for now; Week 10 fetches it from the API.
function App() {
  const jobs = [
    { _id: '1', title: 'Frontend Developer',  company: 'Acme Corp',  location: 'Bangkok',    type: 'Full-time' },
    { _id: '2', title: 'Backend Engineer',    company: 'StartupTH',  location: 'Remote',     type: 'Full-time' },
    { _id: '3', title: 'Junior Web Developer',company: 'Agency BKK', location: 'Bangkok',    type: 'Part-time' },
  ];

  return (
    <div className="app">
      <header>
        <h1>Dev Jobs</h1>
      </header>
      <main>
        <JobList jobs={jobs} />
      </main>
    </div>
  );
}

export default App;
```

```jsx
// frontend/src/main.jsx — entry point (generated by Vite, shown for context)
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App.jsx';
import './index.css';

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

## Try It Yourself

Create a `frontend/` folder in your `job-board-spine` repo. Run `npm create vite@latest . -- --template react` inside it to scaffold a React + Vite project. Delete the boilerplate in `App.jsx` and replace it with your own `JobCard` and `JobList` components. Use the hardcoded jobs array for now — Week 10 will fetch from your Express API. Run `npm run dev` and verify the job list renders at `http://localhost:5173`. Commit with `Week 9: React frontend with hardcoded jobs`. (Week 9)

## The AI Age

**When to use AI for this topic:** Ask AI to scaffold a new React component from a description. Describe the props it should accept, what it should render, and any conditional logic. AI generates clean functional components consistently.

**What AI gets wrong:** AI sometimes generates class components (`class MyComponent extends React.Component`) instead of functional components. Class components are the old API — do not use them in new code. If AI generates a class component, ask it to rewrite as a function.

**Try this prompt in Cursor:**
```
Write a React functional component called JobCard that accepts a job prop
with these fields: title (string), company (string), location (string),
type (string), description (optional string).
- Render title in an <h2>
- Render company, location, and type in a single <p> separated by dots
- Only render description if it exists
- Use className (not class) for CSS classes
- Export as default
```
